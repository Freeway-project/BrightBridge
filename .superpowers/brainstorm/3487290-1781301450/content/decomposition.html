A

Instructor Workspace Redesign HIGHEST IMPACT

Non-tech friendly layout: all 3 TA review forms visible at once (no nav needed), colour-coded status per section, large readable cards. Simple ↔ Full view fully synced — chat/questions/issues appear identically in both modes. Instructor never has to "find" anything.

instructor-workspace simple-full-sync review-data-display
B

Unified Communication Thread

Single conversation visible to TA, instructor, and admin — one source of truth. TA side surfaces instructor questions inline. Admin can participate. Course timeline records every message. No more "where did they ask that?" confusion.

ta-instructor-chat admin-in-loop unified-thread
C

Engagement Tracking

Record which review sections the instructor actually opened/scrolled. Track issue acknowledgments — did they tap "Seen"? Admin dashboard shows: "Manvir opened the course 3× but hasn't reached the Review Matrix." Drives follow-up.

section-view-tracking issue-acknowledgment admin-engagement-view
D

Hierarchy / Leadership Views

Dean, provost, and dept-head have very different needs. Tabs segmented by college → school → department (not a flat tree). Colour-coded levels. Expanded by default for their own scope. Course counts, status breakdowns per unit at a glance.

dean-provost-view tab-segregation expanded-by-default

Recommended build order

A first (instructors are blocked on this) → B next (communication is what A exposes) → C then (tracking only matters once they're using it) → D last (leadership views are read-only, lower urgency)
